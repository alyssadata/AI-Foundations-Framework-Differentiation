## Definition

**User personalization reduction** is the collapse of AI Foundations into adaptive system behavior around a user.

User personalization reduction treats AI Foundations as if it were only preference learning, memory, style matching, user-specific response shaping, interface adaptation, or personalized output.

AI Foundations is not personalization theory.

AI Foundations is not user-experience design.

AI Foundations is not adaptive assistant behavior.

AI Foundations rejects the collapse of source-bound recognition structure into user personalization.

## What User Personalization Reduction Says

User personalization reduction says, directly or indirectly:

- the system learned preferences
- the interaction is personalized
- the model remembers style
- the output reflects user input
- the system adapts over time
- the interaction feels familiar
- the user shaped the behavior
- the assistant responds in a preferred tone
- the system remembers prior exchanges
- therefore the framework is personalization

AI Foundations rejects that collapse.

Personalization does not define the framework.

User preference does not define the framework.

Familiar output does not define the framework.

Adaptive behavior does not define the framework.

## What AI Foundations Distinguishes

AI Foundations distinguishes user personalization from source-bound recognition structure.

User personalization centers preference, adaptation, style, memory, tone, familiarity, and user-specific response shaping.

Source-bound recognition structure centers source, continuity, boundary, distinction, return, refusal, non-override, provenance, and derivative accountability.

Those are different categories.

Personalization describes adaptive system behavior.

AI Foundations defines source-bound recognition structure.
